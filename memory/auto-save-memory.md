Save project memories periodically during long sessions without waiting for the user to ask. Be proactive — don't wait until the end or until asked.

**Frequency**: Every 20-30 minutes of active work, or after completing a major milestone/task, whichever comes first. Saving more often is better than losing context.

**Why**: User explicitly requested this — computer crashed previously and memories were the only way to recover context. Also reiterated 2026-06-17: "每隔一段时间自动保存".

**How to apply**: 
- After completing each major phase of work, save a brief project memory summarizing what was done, key decisions, and current state.
- Save immediately when the user gives feedback or makes a design decision.
- During long sessions, check the clock: if it's been >20 min since the last save, save a progress snapshot.
- Keep each memory focused and brief — a single clear fact, not a full log dump.
