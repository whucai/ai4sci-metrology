Executive Summary

This report reframes the existing project from a Chinese-language scientometrics benchmark into a UTD-targeted IS/Management Science paper. The core move: from "evaluating whether LLMs can reproduce papers" → "under what conditions can organizations trust AI-generated research reproductions?"

The paper proposes a Task-Component Theory of AI Reproducibility Governance — a three-layer framework (Task Type × Evidence Chain Component × Auditability Maturity) that decomposes reproduction quality and detects spurious reproduction that traditional result-level metrics miss.

One-sentence positioning: 本研究提出 AI 复现质量的"任务—组件—治理"理论框架，通过多论文、多模型、多信息条件实验，检验生成式 AI 在计算型研究复现中的能力边界，并证明组件级审计能够识别传统结果指标掩盖的虚假复现。

The Strategic Reframing: From Benchmark to Theory

Problem with Current Framing

The current project narrative ("大模型驱动的计算型论文复现质量评估" + SciSciBench + Reproduction Framework + M0/M1 + retracted-paper-detection) reads as:

  • A Chinese-language scientometrics workshop paper
  • Too many sub-projects without a single contribution chain
  • "We built X, we ran Y, we measured Z" — not a theoretical contribution

The UTD Reframing

Single contribution chain:

This paper proposes a task-component theory of AI reproduction quality; validates it through multi-paper, multi-model, multi-condition experiments; and demonstrates that result-level metrics systematically overestimate reproduction quality while component-level auditing identifies spurious reproduction and traces failures to specific evidence-chain components.

Paper title candidates:

  1. Auditing Scientific Knowledge with Generative AI: A Task-Component Framework for Reproducible Computational Research (IS-leaning, recommended)
  2. When Can AI Agents Reliably Reproduce Computational Studies? Evidence from a Task-Component Benchmark of Scientific Reproducibility (Management Science-leaning)
  3. Can Generative AI Reproduce Computational Research? A Task-Component Theory of AI Reproducibility Governance (theory-leaning)

Three Research Questions (Governance-Oriented)

  • RQ1: Under what information conditions can generative AI agents reproduce computational research faithfully?

    • Maps to: Information condition → AI reproduction performance

  • RQ2: Which components of the research evidence chain create the greatest bottlenecks for AI-based reproduction?

    • Maps to: Task structure → Failure mechanisms

  • RQ3: Can an audit-oriented evaluation framework detect spurious reproductions that would otherwise appear successful?

    • Maps to: Audit framework → Governance value

Five Testable Hypotheses

  • H1 (Information Condition Effect): AI reproduction fidelity increases as the information condition becomes more complete, but the marginal gain is larger for upstream components (data, sample) than for downstream claims.
  • H2 (Component Bottleneck): Errors in sample construction, indicator operationalization, and model specification mediate the relationship between information condition and final result accuracy.
  • H3 (Spurious Reproduction Detection): Result-level evaluation overestimates AI reproduction quality relative to component-level audit evaluation.
  • H4 (Task Complexity Moderation): The performance gap between AI agents is larger for high-ambiguity reproduction tasks than for strict numerical reproduction tasks.
  • H5 (Audit Framework Value): An audit-oriented evaluation protocol reduces false-positive reproduction success by identifying spurious agreement between generated and target results.

The Framework: Task—Component—Governance Model

Layer 1: Task Type (What is being attempted?)

AI reproduction is not a single task but a family of tasks:

Exact ReproductionExact reproduction under identical data conditionsSTRICTImplementation correctness
Robustness ReproductionRobustness reproduction under substitute dataDATA-SUBMechanism consistency
Procedural ReproductionProcedural reproduction without numerical targetsMETHODStructural understanding
Claim RobustnessClaim robustness under external data conditionsCLAIM-ROBUSTExternal validity

Layer 2: Evidence Chain Components (Where does it break?)

A computational paper is not reproduced by matching a final number alone; it is reproduced by reconstructing the evidence chain that links data, operationalization, model estimation, numerical results, and substantive claims.

Data Source → Sample → Indicator → Model → Result Table → Claim

Each component answers a specific evaluation question:

  • Data Source: Did the agent find the correct data?
  • Sample: Did the agent construct the correct sample?
  • Indicator: Did the agent implement the correct indicators?
  • Model: Did the agent estimate the correct model?
  • Result Table: Does the agent's output match the paper's target results?
  • Claim: Does the agent's evidence support the original conclusions?

Layer 3: Auditability Maturity (How much can we trust it?)

The maturity levels operationalize different degrees of organizational trust in AI-generated reproduction outputs:

LevelNameTrust Implication
L0FailureZero trust — cannot execute or completely deviates
L1Executable AttemptCode runs but results are inconsistent — untrusted
L2Partial ReproductionProcess roughly correct — low trust
L3Numerically Verified ReproductionKey numbers match — moderate trust
L4Claim-Consistent ReproductionConclusions supported — high trust
L5Auditable ReproductionFull pipeline traceable, errors explainable — maximum trust

Governance contribution: L0-L5 is not a scoring rubric — it is a trust calibration instrument for research organizations, journals, and labs deploying AI agents in scientific knowledge work.
